# 확장 로드맵

> MVP(persistence + 최소 멀티유저) 이후 GDD의 "미구현 시스템"을 순서대로 구현했다. 아래 1~7은 **구현 완료(✅)** 상태이며, 남은 것은 고도화·확장 항목이다.

## 1. 협업 시스템 (GDD §10) — ✅ 구현 완료
초대 → 수락 → 역할(작곡/작사/편곡/보컬/프로듀싱) 확정 → 발매 시 `contribution%` 기준 자동 수익 분배. `collab_invites` / `song_collaborators` 테이블, `/collab/*` 엔드포인트, 비트메이커의 초대 패널 + 협업 초대함 화면. **남은 고도화**: 동기 공동 편집 세션(현재는 비동기 초대만).

## 2. 회사 시스템 (GDD §12) — ✅ 구현 완료 (기본)
회사 설립 → 연습생 모집 → 트레이닝(커리큘럼 단계) → 그룹 데뷔. `companies`/`trainees`/`groups` 테이블, `/company/*` 엔드포인트, 회사 화면. **남은 고도화**: 월드 투어, 해외 진출, 그룹의 곡 생산 파이프라인 연동, 가수/프로듀서 영입.

## 3. 경제 시스템 심화 (GDD §13) — ✅ 구현 완료 (수익원 세분화)
발매 시 수익원별(스트리밍/공연/광고/팬클럽/앨범판매/라이선스) 세분화 + 지출(제작·보컬) 표시(`songs.revenue_breakdown`, `services/economy.py`). 결과 화면에 수익 내역 노출. **남은 고도화**: 직원 급여·마케팅·장비·사옥 등 상시 지출 원장(ledger), 회사 손익계산 통합.

## 4. 팬 시간 경과 시뮬레이션 (GDD §9) — ✅ 구현 완료 (로그인 배치)
로그인 시 경과일 기준 팬 유입/이탈 + 스트리밍 패시브 수익 정산(`services/fan_simulation.py`, `characters.last_simulated_at`). **남은 고도화**: 발매 후 수일에 걸친 지연 노출(역주행 지속성), 신곡 부스트 감쇠 곡선 등 시간축 모델링.

## 5. 트렌드/라이벌 시스템 — ✅ 구현 완료 (트렌드)
주 단위 유행 장르/무드 결정론적 로테이션(`services/trends.py`), 매칭 시 노출 보너스, 스튜디오 트렌드 배너. **남은 고도화**: 라이벌 아티스트(경쟁 NPC의 능동적 발매·차트 경쟁).

## 6. 온라인 연결 기능 (GDD §11) — ✅ 구현 완료 (콘서트/마켓플레이스)
일정 예약형 콘서트(티켓 판매/수익), 곡 거래 마켓플레이스(등록/구매). `concerts`/`concert_tickets`/`marketplace_listings` 테이블, `/concerts`·`/marketplace` 엔드포인트, 온라인 화면. **남은 고도화**: 콘서트 실시간 동시 접속(WebSocket), 아티스트 영입 계약 플로우.

## 7. 업적/마일스톤 시스템 — ✅ 구현 완료
10종 업적(데뷔/첫 성공/대박/다작/팬 규모/자산/명성/천재 이벤트/역주행), 발매·로그인 시 판정(`services/achievements.py`), 스튜디오 업적 패널 + 발매 결과 화면 달성 배너.

## 8. 팔로우 영속화 — ✅ 구현 완료
`follows` 테이블 + `/community/follows`. 새로고침해도 유지, optimistic update.

## 남은 확장 항목 (미착수)

### A. 차트/노출 알고리즘 고도화
현재 차트는 점수 합산 정렬. 순위 변동 히스토리, 신곡 부스트 가중치, 역주행 지속 판정을 별도 스코어링 레이어로 분리.

### B. 실시간 기능 (WebSocket)
콘서트 동시 접속, 실시간 차트 경쟁 등. `server-architecture.md §1`에서 자리 예약됨.

### C. 회사 시스템 심화
월드 투어, 해외 진출, 그룹 곡 생산, 아티스트 영입 계약.

### D. 경제 원장(ledger)
상시 지출(급여/마케팅/장비/사옥) 추적, 회사 손익계산 통합.

### E. 모바일 대응
`ui-ux.md §4`의 데스크톱 우선 결정 재검토 — 비트메이커 터치 UX 재설계가 선행 조건. (PWA 설치는 이미 지원.)

### F. 배포
Vercel(프론트) + Railway/Render(백엔드 + Postgres). 현재는 로컬 SQLite로만 검증됨.
