# 확장 로드맵

> MVP(persistence + 최소 멀티유저) 이후 GDD의 "미구현 시스템"을 순서대로 구현했다. 아래 1~7은 **구현 완료(✅)** 상태이며, 남은 것은 고도화·확장 항목이다.

## 1. 협업 시스템 (GDD §10) — ✅ 구현 완료
초대 → 수락 → 역할(작곡/작사/편곡/보컬/프로듀싱) 확정 → 발매 시 `contribution%` 기준 자동 수익 분배. `collab_invites` / `song_collaborators` 테이블, `/collab/*` 엔드포인트, 비트메이커의 초대 패널 + 협업 초대함 화면. **남은 고도화**: 동기 공동 편집 세션(현재는 비동기 초대만).

## 2. 회사 시스템 (GDD §12) — ✅ 구현 완료 (기본)
회사 설립 → 연습생 모집 → 트레이닝(커리큘럼 단계) → 그룹 데뷔. `companies`/`trainees`/`groups` 테이블, `/company/*` 엔드포인트, 회사 화면. **남은 고도화**: 월드 투어, 해외 진출, 그룹의 곡 생산 파이프라인 연동, 가수/프로듀서 영입.

## 3. 경제 시스템 심화 (GDD §13) — ✅ 구현 완료 (수익원 세분화)
발매 시 수익원별(스트리밍/공연/광고/팬클럽/앨범판매/라이선스) 세분화 + 지출(제작·보컬) 표시(`songs.revenue_breakdown`, `services/economy.py`). 결과 화면에 수익 내역 노출. **남은 고도화**: 직원 급여·마케팅·장비·사옥 등 상시 지출 원장(ledger), 회사 손익계산 통합.

## 4. 팬 시간 경과 시뮬레이션 (GDD §9) — ✅ 구현 완료 (로그인 배치)
로그인 시 경과일 기준 팬 유입/이탈 + 스트리밍 패시브 수익 정산(`services/fan_simulation.py`, `characters.last_simulated_at`). **남은 고도화**: 발매 후 수일에 걸친 지연 노출(역주행 지속성), 신곡 부스트 감쇠 곡선 등 시간축 모델링.

## 5. 트렌드/라이벌 시스템 — ✅ 구현 완료 (트렌드)
주 단위 유행 장르/무드 결정론적 로테이션(`services/trends.py`), 매칭 시 노출 보너스, 스튜디오 트렌드 배너. **남은 고도화**: 라이벌 아티스트(경쟁 NPC의 능동적 발매·차트 경쟁).

## 6. 온라인 연결 기능 (GDD §11) — ✅ 구현 완료 (콘서트/마켓플레이스)
일정 예약형 콘서트(티켓 판매/수익), 곡 거래 마켓플레이스(등록/구매). `concerts`/`concert_tickets`/`marketplace_listings` 테이블, `/concerts`·`/marketplace` 엔드포인트, 온라인 화면. **남은 고도화**: 콘서트 실시간 동시 접속(WebSocket), 아티스트 영입 계약 플로우.

## 7. 업적/마일스톤 시스템 — ✅ 구현 완료
10종 업적(데뷔/첫 성공/대박/다작/팬 규모/자산/명성/천재 이벤트/역주행), 발매·로그인 시 판정(`services/achievements.py`), 스튜디오 업적 패널 + 발매 결과 화면 달성 배너.

## 8. 팔로우 영속화 — ✅ 구현 완료
`follows` 테이블 + `/community/follows`. 새로고침해도 유지, optimistic update.

## 9. DAW급 음악 제작 도구 (핸드오프 v2 §1-A) — ✅ 1~3번 구현 완료
프로토타입 수준의 스텝 시퀀서 + 기본 오실레이터를 "실제 프로듀서용 미니 DAW"에 가깝게 재설계. §1-A가 제시한 9개 항목 중 1~3번을 이번에 구현하고 4~9번은 문서 자체의 권장에 따라 명시적으로 로드맵으로 미뤘다 — `frontend-architecture.md §7` 참고.

- **1. 타임라인**: `Timeline.jsx` — 클립 룰러(드래그 재정렬/리사이즈/삭제) + 4트랙 활동량 레인 + 재생 헤드.
- **2. 정밀 피아노 롤**: 드래그 페인트(노트 길이) + 벨로시티(0-127) + 휴머나이즈. 다음성은 데이터 모델 한계로 범위 밖.
- **3. 더 나은 음색**: 신스 파라미터 고도화(레이어드 드럼, 필터 엔벨로프 베이스, 코러스 피아노, PluckSynth 기타) + 마스터 컴프레서/리미터. 샘플 기반 악기 대신 문서 자체가 허용한 대안 경로.

**8. 실제 녹음 — ✅ 구현 완료 (녹음실)**
마이크 입력으로 보컬을 직접 녹음하고 서버에 저장한다. `lib/audio/recorder.js`(MediaRecorder + 입력 레벨 미터), `RecordingStudio.jsx`(`/recording`), `vocal_recordings` 테이블 + `/recordings` API. 오디오 바이트는 `backend/uploads/vocals/`에 두고 DB엔 메타데이터만 저장한다. 테이크는 작업 중인 곡에 연결할 수 있고, 반주와 함께 녹음/재생할 수 있다. 파일명은 항상 서버가 UUID로 생성하며(경로 조작 방지), 오디오 MIME만 허용하고 10MB로 제한한다. 인증이 필요한 오디오라 `<audio src>` 대신 blob을 fetch해 object URL로 재생한다.

**보너스: 오토튠(음정 보정) — ✅ 구현 완료**
녹음된 테이크를 오프라인 처리해 스케일에 맞게 음정을 보정한다 (`lib/audio/autotune.js`, 녹음실의 오토튠 패널). 키·스케일(메이저/마이너/펜타토닉/크로매틱)·보정 강도(0~100%)를 고르고, 미리 들어본 뒤 새 테이크로 저장한다 — 원본은 보존된다.
- 검출: 자기상관 + 정규화 제곱차, 포물선 보간으로 서브샘플 정밀도. RMS·신뢰도 게이트로 무음/자음 구간은 건드리지 않음.
- 보정: 2탭 크로스페이드 딜레이라인 기반 시간가변 피치 시프트. 블록별 그레인 리샘플 방식은 각 그레인이 입력 타임라인에 재동기화되어 원본 음정이 그대로 복원되는 문제가 있어 채택하지 않음.
- 안정화: 피치 트랙 중앙값 필터 + 5센트 히스테리시스 — 스케일 두 음의 정확히 중간에 있는 음이 블록마다 목표를 바꿔 워블링되는 것을 방지.
- **검증**: 38~99센트 어긋난 합성 톤 6종이 모두 스케일 음 **4센트 이내**로 안착. 강도 0/25/50/100%가 46.6→34.8→23.4→-1.3센트로 선형 반영. WAV 인코딩 왕복 후에도 음정 유지.
- **한계**: 단선율(모노포닉) 전용 — 화음이 섞인 소스는 검출이 실패해 보정하지 않는다. 실시간(녹음 중) 보정이 아니라 녹음 후 처리 방식이다.

**남은 4~7, 9번 (미착수, 다음 DAW 마일스톤)**:
- **4. 믹싱 콘솔**: 채널별 멀티밴드 EQ, 컴프레서(threshold/ratio/attack/release), 사이드체인, 컨볼루션/정교한 알고리즘 리버브.
- **5. 오토메이션**: 볼륨/팬/이펙트 파라미터를 시간에 따라 변화시키는 오토메이션 레인.
- **6. MIDI 임포트/익스포트**: `.mid` 파일 불러오기/내보내기.
- **7. 오디오 렌더링/익스포트**: 완성곡을 실제 WAV/MP3로 내보내기 (현재는 세션 재생만 가능). 녹음 보컬 + 반주를 한 파일로 믹스다운하는 것도 여기 포함.
- **9. DAW 표준 UX**: undo/redo, 스페이스바 재생/정지 등 업계 표준 단축키, 확대/축소 가능한 타임라인.
- **부가**: 다음성(한 트랙 내 동시 다중 노트) — 현재 스텝당 값 하나 배열 구조를 노트 단위 구조로 바꾸지 않는 한 불가, 백엔드 스키마 변경이 필요해 별도 결정 필요.

**녹음 관련 남은 것**: 녹음한 보컬은 아직 발매곡에 믹스되지 않는다 — 커뮤니티 피드는 여전히 반주(패턴)만 재생하므로, 다른 유저가 내 보컬을 들으려면 7번(믹스다운) 또는 피드 재생에 보컬 트랙을 얹는 작업이 필요하다.

## 남은 확장 항목 (미착수)

### A. 차트/노출 알고리즘 고도화
현재 차트는 점수 합산 정렬. 순위 변동 히스토리, 신곡 부스트 가중치, 역주행 지속 판정을 별도 스코어링 레이어로 분리.

### B. 실시간 기능 (WebSocket)
콘서트 동시 접속, 실시간 차트 경쟁 등. `server-architecture.md §1`에서 자리 예약됨.

### C. 회사 시스템 심화
월드 투어, 해외 진출, 그룹 곡 생산, 아티스트 영입 계약.

### D. 경제 원장(ledger)
상시 지출(급여/마케팅/장비/사옥) 추적, 회사 손익계산 통합.

### E. 모바일 대응
`ui-ux.md §4`의 데스크톱 우선 결정 재검토 — 비트메이커 터치 UX 재설계가 선행 조건. (PWA 설치는 이미 지원.)

### F. 배포
Vercel(프론트) + Railway/Render(백엔드 + Postgres). 현재는 로컬 SQLite로만 검증됨.
